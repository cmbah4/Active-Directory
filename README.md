# Active Directory Account Lockout Investigation

## Overview
This project demonstrates a basic Active Directory-style investigation using Windows security logs. The goal was to simulate failed login attempts and analyze the resulting events to identify potential brute-force activity.

---

## Lab Setup
- Operating System: Windows 10 Virtual Machine  
- Tools Used: Event Viewer  
- Environment: Local user account management (Computer Management)

---

## Scenario
A user account (jdoe) was created and multiple failed login attempts were intentionally generated by entering incorrect credentials. This simulated a potential brute-force attack.

## What Happened
Multiple failed logins were observed on jdoe, indicating a potential brute force attempt.

---

## Recommendations
To prevent unauthorized attackers from accessing user accounts, I recommend enabling two-factor authentication. In addition, I recommend enforcing an account lockout policy to lock the account after repeated failures and prevent the attacker from trying an unlimited number of times.

---

## Conclusion
This lab demonstrated how failed login attempts are recorded in Windows Security logs and how repeated failures can indicate suspicious activity. It also reinforced the importance of monitoring authentication events and implementing security controls to prevent unauthorized access.
